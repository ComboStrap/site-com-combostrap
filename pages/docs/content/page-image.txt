====== ComboStrap UI - Page Image Component ======


===== About =====
''page-image'' is a [[content|content component]] that chooses and renders one of the [[docs:page:image|page images]] (illustration, thumbnail) defined for a page.


<note>
This component is generally used in a [[docs:templating:iterator|template iteration]] to create illustrative image for [[docs:block:block|block component]] (mainly for [[docs:block:card|cards]])
</note>


The generated image inherits all [[docs:content:image|images features]] such as lazy loading and responsiveness.

===== Example =====

==== The original page image ====

This example shows the [[docs:page:image|page image]] of this page (path: ''<template>$path</template>'') - We have set it to the image of the surfer below.

<webcode name="Default" frameborder="0">
<code dw>
<page-image path=":docs:content:page-image" />
</code>
</webcode>

==== A page image with a 4:3 ratio ====

Note that the image has an intrinsic ratio of ''16:9'', we can change it to a ratio of ''4:3'' with the below markup.

<webcode name="Default" frameborder="0">
<code dw>
<page-image path=":docs:content:page-image" ratio="4:3"/>
</code>
</webcode>

===== Syntax =====

<code dw>
<page-image path="path" ratio="ratio" color="color"/>
</code>
where:
  * ''path'' is the [[docs:page:path|path]] of a page
  * ''ratio'' is the ratio of the image (default to original):
      * ''16:9''
      * ''4:3''
      * ''1:1''
  * ''color'' is a [[docs:styling:color|color name]]. This attribute applies only on [[docs:content:svg|svg]] page image, particularly on [[docs:content:icon|icon]].

When the ''ratio'' is set, the algorithm will:
  * select the image in the set of [[docs:page:image|page image]] with the most equivalent ratio
  * [[docs:content:image#cropping|crop]] the image to fit the ratio

<note>
  * The ratio is important in a [[docs:templating:iterator|template iteration]] because it permits to get the same height for all images. This way the layout will get a consistent and peaceful visual.
  * For icon page image, you should be able to use all [[docs:content:icon|icon parameters]]
</note>

===== Support =====

==== Icon Support ====

When the image is an [[docs:content:icon|icon]], the icon is zoomed out to 3 times and centered.

<webcode name="Default" frameborder="0">
<code dw>
<page-image path="docs:templating:sql" ratio="16:9" color="pink" width="198px"/>
</code>
</webcode>


==== Card Support ====


A page-image is supported as illustrative image for a [[docs:block:card|card]]

Example a card with data of this page (thanks to the [[docs:templating:template|template component]])
<webcode name="Default" frameborder="0">
<code dw>
<template>
    <card clickable width="300">
        <page-image path="$path" ratio="16:9"/>
        === $h1 ===
        $description [[$path| ]]
    </card>
</template>
</code>
</webcode>
