====== ComboStrap UI - Page Image Component ======


===== About =====
''page-image'' is a [[content|content component]] that chooses and renders one of the [[docs:page:image|page images]] (illustration, thumbnail) defined for a page.




===== Example =====

==== The original page image ====

This example shows the [[docs:page:image|page image]] of this page. We have set it to the image of the surfer below.

<webcode name="Default" frameborder="0">
<code dw>
<page-image/>
</code>
</webcode>
==== How to get a serie of images with the same dimension  ? ====


In a [[docs:templating:iterator|fragment iteration]] where the width of each element is set, the ratio permits to get the same image aspect-ratio as well as the same dimension for all images. The visual rendering is therefore consistent and peaceful.


For instance, the page image has an intrinsic ratio of ''16:9'', we can change it to a ratio of ''4:3'' with the below markup and constraints the height 

<webcode name="Default" frameborder="0">
<code dw>
<page-image ratio="4:3" height="200"/>
</code>
</webcode>




===== Syntax =====

<code dw>
<page-image 
    default="fromParent"
    path="path"
/>
</code>
where 
  * ''path'' is the [[docs:page:path|page path]] to show the image for. Default to the actual page path or the [[:docs:templating:iterator|path given by the iterator]] 
  * ''default'' determines where to start in the [[#default image search algorithm]]. ''fromParent'' will skip the first step and not select the first image of the page (used mostly in a [[:slot_main_header|main header]] to not show the same image twice).


<note>
  * All [[docs:content:image|image attributes]] such ''width'' / ''height'' / ''ratio'' and ''color'' and more can be applied.
  * For an [[docs:content:icon|icon]], the default color is the [[docs:app:color|primary color]] if set
</note>

===== Default Image Search Algorithm =====

When searching a default image, the algorithm searches:
  * the first image in the actual page
  * then if not found, a image in the [[:docs:page:system|parent pages]]
  * until the [[:docs:app:home|home Page]] where the [[:docs:app:logo|logo]] is chosen as page image.
 
===== Image selection =====

When the ''ratio'' is set, the algorithm will:
  * select the image in the set of [[docs:page:image|page image]] with the most equivalent ratio
  * [[docs:content:image#cropping|crop]] the image to fit the ratio


===== Support =====

==== Icon Support ====

When the image is an [[docs:content:icon|icon]], the icon is zoomed out to 3 times and centered.

<webcode name="Default" frameborder="0">
<code dw>
<page-image path="docs:templating:sql" ratio="16:9" color="pink" width="198" border-color="pink"/>
</code>
</webcode>


==== Card Support ====


A page-image is supported as illustrative image for a [[docs:block:card|card]]

Example a card with data of this page with [[:docs:templating:variable|variable]]
<webcode name="Default" frameborder="0">
<code dw>
<card clickable width="300">
    <page-image ratio="16:9"/>
    === $h1 ===
    $description [[$path| ]]
</card>
</code>
</webcode>
