====== ComboStrap UI - Page Explorer  ======



===== About =====
A ''page-explorer'' is a [[docs:navigation:navigation|navigational component]] that creates a navigation pane in the form of:
  * a ''list'' 
  * or a collapsible ''tree'' (ie clickable levels of folder hierarchy)


This component is mostly used in a [[docs:layout:page:side_slot|side slot]] or [[docs:layout:page:sidekick_slot|sidekick slot]]


===== Example =====

==== Default List ====

A default ''page-explorer list'' pane can be asked with this minimal markup (the ''ns'' ([[doku>namespaces|namespace]]) and [[docs:styling:dimension|width]] options are just cosmetic and could be deleted)

<webcode name="DefaultList" >
<code dw>
<page-explorer ns=":release" width="300px"></page-explorer>
</code>
and will look like this:
</webcode>


==== Default Tree ====

A ''page-explorer tree'' does not need any template definition. 


<webcode name="DefaultTree" >
  * with a minimal markup (The ''ns'' option is a cosmetic and can be deleted)
<code dw>
<page-explorer tree ns=":docs:layout"></page-explorer>
</code>
  * a page-explorer tree will look like this:
</webcode>

==== Custom List ====

<webcode name="Default" frameborder=0 >
<code dw>
<page-explorer color="blue" height="250px" width="400" scroll="toggle">
    <parent>[[$path|<icon name="arrow-left-box"/> ... <pipeline>"$h1" | cut("-","2-") | trim()</pipeline>]]</parent>
    <page>[[$path|<icon name="file"/> <pipeline>"$h1" | cut("-","2-") | trim()</pipeline>]]</page>
</page-explorer>
</code>
where the ''page'' template is composed of:
  * an [[docs:content:icon|icon]]
  * a [[docs:content:link|link]] without [[docs:styling:underline|underline]]
  * and a [[docs:templating:pipeline|pipeline]] to modify the [[docs:page:h1|heading 1]] of the page.
</webcode>

==== Custom Collapsible Tree ====


Example:

<webcode name="Default">
<code dw>
<page-explorer tree ns=":docs:layout">
    <home>[[$path|<icon name="home" color="#0000004d" width="20"/> Overview]]</home>
    <ns><pipeline>"$h1" | cut("-","2-") | trim() </pipeline></ns>
    <page>[[$path|<icon name="file" color="#0000004d" width="20"/>  <pipeline>"$h1" | cut("-","2-") | trim() </pipeline>]]</page>
</page-explorer>
</code>
where the ''page'' template is composed of:
  * an [[docs:content:icon|icon]]
  * a [[docs:content:link|link]]
  * and a [[docs:templating:pipeline|pipeline]] to modify the [[docs:page:h1|heading 1]] of the page.

It produces:
</webcode>



===== Syntax =====

<code dw>
<page-explorer type ns="namespacePath">
  <page>template</page>
  <namespace>template</namespace>
  <home>template</home>
  <parent>template</parent>
</page-explorer>
</code>
where:
  * the attributes are:
      * [[docs:styling:type|type]] that defines the layout and may be one of this two value:
          * [[#list|list]] to create a list navigation pane (the default)
          * [[#tree|tree]] to create a tree navigation pane
      * ''ns'' that defines the ''namespace''. The default  value is the namespace of:
          * the rendered page for a [[#list]]
          * the physical page for a [[#tree]].
  * the sub-elements ''page'',''namespace'', ''home'' and ''parent'' define [[#template|templates]] for each ''page-explorer'' node.


''page-explorer'', ''home'', ''page'', ''parent'' and ''namespace'' are all [[docs:marki|superset HTML component]].
==== Templates ====

The templates define the output of each node in the ''list'' or ''tree'' layout:
    * ''page'' defines a [[docs:templating:template|template]] for each page in the namespace
    * ''home'' defines a [[docs:templating:template|template]] for the [[docs:type:home|home page of the namespace]]
    * ''namespace'' defines a [[docs:templating:template|template]] for each child namespace of the actual namespace 
    *  ''parent'' defines a [[docs:templating:template|template]] for the parent namespace of the actual namespace (used only in the [[#list|list layout]])
  

The below image shows you the mapping between the tag and their location in a ''list''  layout.

{{docs:block:page-explorer-list-name-to-ui-region.png|}}

The below image shows you the mapping between the tag and their location in a ''tree'' layout.

<note>
If a sub-element (''home'', ''parent'', ''namespace'') is defined without any template (ie empty content), they are not shown.
</note>

==== Default template ====

  * if the ''page'' tag is is not defined, the below default ''page'' template is used 
<code html>
[[$path|<pipeline>"$name" | replace("_"," ") | capitalize()</pipeline>]]
</code>
  * if the ''namespace'' tag is not defined, a default namespace template is used
<code html>
<!-- For a list -->
[[$path|<icon name="folder"/> <pipeline>"$name" | replace("_"," ") | capitalize()</pipeline>]]
<!-- For a tree -->
<pipeline>"$name" | replace("_"," ") | capitalize()</pipeline>
</code>
  * if the ''parent'' tag is not defined, a default parent template is used
<code html>
[[$path|<icon name="arrow-left-box"/> ... <pipeline>"$name" | replace("_"," ") | capitalize()</pipeline>]]
</code>
  * if the ''home'' tag is is not defined, the ''page'' template is used  


===== Support =====

==== Performance consideration with the tree pane ====
The [[#tree|page-explorer tree]] is recreated every time that a subpage is modified. If your tree has a lot of page, it may perform poorly:
  * while editing 
  * and for the first viewer
