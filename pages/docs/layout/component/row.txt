====== ComboStrap UI - Grid (Row and Cell) ======


===== About =====
''row'' is a component that permits to layout ''cells'' on the horizontal axis.

If a cell does not pass horizontally, it goes to a new line in a centered position.

A row layout makes also all cells readable from the start from a mobile device.

==== Generated via the iterator ====

You can use the ''row'' component in a [[:docs:templating:iterator|Iterator]] to generate a list of cells.

<webcode>
<code dw>
<iterator>
    <data>select from descendants where depth = 0 order by path asc limit 5 </data>
    <row>
        <template>
            <cell clickable>
                <page-image path="$path" width="8rem" align="center" />
                == $title ==
               <pipeline> "$description" | head(100,"...") </pipeline>
               [[$path| ]]
            </cell>
       </template>
    </row>
</iterator>
</code>
  * Output:
</webcode>

===== Syntax =====
<code dw>
<row type>
   <cell width="ratioValue"></cell>
   <cell></cell>
</row>
</code>
where:
  * ''type'' may be the value:
  * ''width'' may be a rationale value. ie a number with one of the following unit
      * a percentage: ''%''
      * or a fraction: ''fr''

^ Type ^ Description ^ Default for ^
| ''auto'' | each cell width will automatically calculated to be greater than the minimum width of ''300'' | loss grid | 
| ''fit'' | each cell will takes their natural width . Note that for cells with text and without width, we set a width of ''300'' (because the natural width of text is the whole line until the end of the paragraph) | contained grid |
| ''none'' | the content will just shrink without overflow | |
| ''width'' | the width is defined explicitly on the cells | cell with width |


<note>
  * You can use all [[docs:styling:styling|styling attributes]].
  * The dimension of the the cells are not uniform, this is why a [[docs:styling:background|background]] cannot be used inside a row.
</note>
