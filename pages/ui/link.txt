====== ComboStrap UI - Link ======

===== About =====
''link'' is  a [[ui|ComboStrap UI component]] that enhances the [[doku>wiki:syntax#links|link]] syntax by allowing any content inside (by default on the [[#multiLinesLink|same line]]). Dokuwiki allows only text and media.


<note>
You can therefore have an [[#icon|icon link]]
</note>

<badge>Since [[release:release#version_112|version 1.12]]</badge>
===== Articles Related =====
{{backlinks>.}}



===== Type =====

==== Internal ====

An ''internal'' link is a link to another page. If the target page does not exist, the link is shown in ''red''.

  * Example:
<webcode name="Default" frameborder=0>
<code dw>
Links to [[link|existing pages]] are shown in a different style from [[nonexisting|nonexisting]] ones.
</code>
  * Result:
</webcode>


==== External ====

An ''external'' link is a link that begins with ''http'' or ''https''. 

  * Example:
<webcode name="Default" frameborder=0>
<code dw>
External link to our protected page [[https://combostrap.com/quality/low_quality_example|low quality page]] 
</code>
  * Result:
</webcode>
==== Interwiki ====

An ''interwiki'' page is a shortcut syntax that permits to create rapidly a link to another website. The host is replaced by a ''wikiName'' (ie website name)

Syntax:
<code html>
wikiName>path?query#fragment
</code>

For instance:
  * to [[doku>|Dokuwiki]] syntax
<webcode name="Default" frameborder="1">
<code dw>
Interwiki: [[doku>Interwiki]]
</code>
  * to [[doku>|Dokuwiki]] Result
</webcode>
  * to [[wp>|wikipedia]] syntax
<webcode name="Default" frameborder="1" >
<code dw>
The ''publication'' article link can be created with this syntax: [[wp>publication]]
</code>
  * to [[wp>|wikipedia]] result
</webcode>

===== Configuration =====

==== useDokuwikiLinkClassName ====
[[doku>|Dokuwiki]] sets class names (''wikilink1'',''wikilink2'',''urlextern'',...) on the link to differentiate them. Unfortunately, this class names have styling rules that are in conflict with [[styling:bootstrap|Bootstrap]].

To resolve this issue, we have set other class name by link type that a web designer can use to style the links.

If you wish to get the Dokuwiki class name on your links, you can get them by enabling the [[:configuration|useDokuwikiLinkClassName configuration]].


==== disableLink ====
If you don't want that ''Combostrap'' changes the default link behavior, you can disable it with the ''disableLink'' [[:configuration|configuration]].

''ComboStrap'' makes use of the context to output the correct bootstrap formatting. If you disable it, the page rendered will not be [[strap:bootstrap|fully BootStrap compliant]].


==== enableMultiLinesLink ====
By default, a link is allowed only on one line to avoid strange behavior.

For instance, when you forget to close a combostrap link with ''%%]]%%'' because the link allows any syntax inside, an error may appear at the end of the document (such as the double forward slash ''%%//%%'' of an external link ''https://example.com'' may be seen as the //italic / emphasis syntax//)

Most of the links are written on one line and to avoid any strange behavior, this component does not allow by default link to be written on more than one line.

If you still want to enable this component on more than one line, you can check the ''enableMultiLinesLink'' [[:configuration|configuration]].
