====== ComboStrap UI - Page Explorer  ======



===== About =====
A ''page-explorer'' is a [[navigation|navigational component]] that creates a list of page

[[doku>namespaces|namespace (ie directory)]] in a [[content-list|list fashion]] [[toc|table of content (TOC)]] for a ''namespace''. It lays out all pages of a 

This component is mostly used in a [[strap:side_slot|side slot]] or [[strap:sidekick_slot|sidekick slot]]


A navigation pane on the left into clickable levels of folder hierarchy
===== Articles Related =====
{{backlinks>.}}

===== Syntax =====

<code dw>
<page-explorer ns="current">
  <page>$id - $title</page>
  <namespace>$id - $title</namespace>
  <home>$id - $title</home>
</page-explorer>
</code>
where:
  * the sub-element are:
    * ''page'' defines a [[template|template]] for each page in the namespace
    * ''home'' defines a [[template|template]] for the home page of the namespace
    * ''namespace'' defines a [[template|template]] for each child namespace of the actual namespace 
  * the attribute are:
      * ''ns''. It defines the ''namespace'' (Default to the ''namespace'' of the current page)
  


<note >
''page-explorer'', ''home'', ''page'' and ''namespace'' are all [[styling:html|superset HTML component]].
</note>

===== Example =====

==== List ====

<webcode name="Default" frameborder=0 width=100% scrolling=yes externalResources="," renderingMode=story >
<code dw>
<page-explorer color="info" height="200px" scroll="toggle">
    <home>[[$id|$title]]</home>
    <page>
        <icon name="file"/>[[$id|<pipeline>"$h1" | replace("ComboStrap UI - ","")</pipeline>]]
    </page>
</page-explorer>
</code>
where the ''page'' template is composed of:
  * an [[icon|icon]]
  * a link
  * and a [[pipeline|pipeline]] to modify the [[semantic:h1|heading 1]] of the page.
</webcode>

==== Collapsible Tree ====

<webcode name="Default" frameborder=0 width=100% scrolling=yes externalResources="," renderingMode=story >

<page-explorer tree ns=":">
    <home>[[$id|$title]]</home>
    <page>
        <icon name="file"/>[[$id|<pipeline>"$h1" | cut("-",2) | trim() </pipeline>]]
    </page>
</page-explorer>
</code>
where the ''page'' template is composed of:
  * an [[icon|icon]]
  * a link
  * and a [[pipeline|pipeline]] to modify the [[semantic:h1|heading 1]] of the page.
</webcode>
